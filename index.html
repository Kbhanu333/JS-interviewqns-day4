Q3. What is callback hell?

It is a function that will be executed after the execution of its parent function. It also forms pyramid like structure that is difficult to understand and debug. We should avoid using callback hell as much as possible.

        function print(){
            setTimeout(() => {
                console.log(5)
                setTimeout(() => {
                    console.log(4)
                    setTimeout(() => {
                        console.log(3)
                        setTimeout(() => {
                            console.log(2)
                            setTimeout(() => {
                                console.log(1)
                            }, 1000);
                        }, 2000);
                    }, 3000);
                }, 4000);
            }, 5000);
        }
        print()

Q4. What are promises and why do we need them?

Promises is like heaven for callback hell. It is introduced in ES6 version to handle asynchronous operations in an efficient way. to debug and manage code we need promises.


Promise allows us to write cleaner code and allows error to pass down the chain and can be handled 
        const printNumber = (time, value) =>{
            return new Promise((resolved, rejected) => {
                setTimeout(() => {
                    console.log(value)
                    resolved();
                }, time);
            })
        }
        printNumber(5000, 5)
            .then(() => printNumber(4000, 4))
            .then(() => printNumber(3000, 3))
            .then(() => printNumber(2000, 2))
            .then(() => printNumber(1000, 1))

Q5. What is promise chaining?

Multiple use of `.then` method will leads to promise chaining.
